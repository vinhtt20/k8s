🎬 Animation 6: ClusterIP vs Headless Service

Cùng 1 set Pod, 2 loại Service trả về DNS rất khác nhau. Bấm "Query DNS" trên cả 2 panel để so sánh.

🅰️ ClusterIP Service

spec:
  type: ClusterIP
  clusterIP: 10.96.0.10 # IP ảo
  selector:
    app: db
DNS query: nslookup my-db.svc
Server: kube-dns
Address: 10.96.0.10 (Service IP)
→ chỉ trả về 1 IP duy nhất (Service)
⚙ Service
10.96.0.10
📦 Pod 1
10.1.1.5
📦 Pod 2
10.1.1.6
📦 Pod 3
10.1.1.7

🅱️ Headless Service

spec:
  type: ClusterIP
  clusterIP: None # ← khác biệt!
  selector:
    app: db
DNS query: nslookup my-db.svc
Server: kube-dns
Address: 10.1.1.5 (Pod 1)
Address: 10.1.1.6 (Pod 2)
Address: 10.1.1.7 (Pod 3)
→ Trả về MỌI Pod IP — client tự chọn!
📦 Pod 1
10.1.1.5
📦 Pod 2
10.1.1.6
📦 Pod 3
10.1.1.7
Khi nào dùng Headless? Khi client cần biết từng Pod cụ thể — ví dụ:
  • StatefulSet (Kafka, Cassandra, MongoDB replica set) — mỗi Pod có vai trò khác nhau, client phải kết nối đúng Pod.
  • Service discovery tự code trong app (Spring Cloud, gRPC client-side LB).
  • Khi không muốn tầng load balance của Service xen vào (vd: gRPC streaming connection).
