📏 So sánh "độ dài" đường đi Càng nhiều hop → latency càng cao

📞 ClusterIP (3 hops)
~0.2 ms · 3 hops · trong cluster
📋 Headless (2 hops)
~0.15 ms · 2 hops · DNS + direct
📮 ExternalName (2 hops)
~5-50 ms · DNS + đi ra Internet
🚪 NodePort (5 hops)
~1-10 ms · 5 hops · external + SNAT
🛎 LoadBalancer (6 hops)
~2-15 ms · 6 hops · qua Cloud LB
💡 Insight: Headless và ClusterIP nhanh nhất vì không qua tầng external. NodePort/LoadBalancer chậm hơn vì phải đi qua mạng public. ExternalName nhanh hay chậm tùy backend ngoài phản hồi như nào.

🌐 Scenario: Mobile App gọi API public

User dùng app mobile, mở app → API call đến api.myapp.com. Cần:

  • IP public ổn định
  • HTTPS termination
  • Hỗ trợ traffic từ Internet
  • Auto health check, fail-over

🏆 Đề xuất: LoadBalancer (kèm Ingress để tiết kiệm)
💡 Lý do: Đây là use case kinh điển của LoadBalancer. Cloud LB sẽ gắn IP public, terminate HTTPS, và route traffic vào cluster. Trong production thật, bạn nên đặt 1 Ingress Controller phía sau LoadBalancer rồi route nội bộ qua ClusterIP — vừa tiết kiệm vừa quản lý dễ.

📞 ClusterIP vs 🛎 LoadBalancer

ClusterIP
• Chỉ trong cluster
• Miễn phí
• 3 hops
• Cho microservices
LoadBalancer
• Truy cập từ Internet
• $18/tháng/LB
• 6 hops
• Cho user public

🔑 Khi nào chọn? Dùng ClusterIP cho mọi giao tiếp nội bộ. Chỉ khi cần expose ra Internet mới dùng LoadBalancer.

📞 ClusterIP vs 📋 Headless

ClusterIP
• DNS trả 1 IP
• iptables load balance
• Pod ẩn danh
• Cho stateless app
Headless
• DNS trả NHIỀU IP
• Client tự chọn
• Pod có DNS riêng
• Cho StatefulSet

🔑 Khi nào chọn? ClusterIP khi Pod ngang hàng. Headless khi mỗi Pod có vai trò khác (master/replica).

🚪 NodePort vs 🛎 LoadBalancer

NodePort
• Dùng IP của Node
• Port 30000-32767
• Miễn phí
• Cho dev/test
LoadBalancer
• IP public từ cloud
• Port chuẩn 80/443
• Tốn phí cloud
• Cho production

🔑 Khi nào chọn? NodePort cho dev. LoadBalancer cho production. On-prem? → cài MetalLB.

📞 ClusterIP vs 📮 ExternalName

ClusterIP
• Có Pod thật trong cluster
• Có IP ảo, có endpoints
• iptables DNAT
• Cho service nội bộ
ExternalName
• KHÔNG có Pod
• Chỉ là DNS CNAME
• Trỏ ra ngoài cluster
• Cho RDS, managed DB

🔑 Khi nào chọn? Backend trong cluster → ClusterIP. Backend ngoài cluster (cloud DB) → ExternalName.
